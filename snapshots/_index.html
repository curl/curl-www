docs/RELEASE-TOOLS.md in the snapshots shows the tooling needed to reproduce the snapshot tarball. This is not necessarily the same set of tools+versions as official curl releases use, because these snapshots are generated automatically on a different host.

docs/tarball-commit.txt in the snapshots shows the commit hash that was the HEAD at the time the tarball was made.
